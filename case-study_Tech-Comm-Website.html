Duration 6 Weeks (March - May 2020)

Our Problem Statement

The technical communication department at Iowa State is struggling to increase the number of students. The technical communication website is one of the main ways they market to students, but it has a number of visual, written, and usability issues that make it unfriendly to prospective students.

Audience

  1. Prospetive students looking to learn more about the technical communication major or minor
  2. Current technical communication students looking for a information about classes

Business Goal

  1. Convince more students to declare a technical communication major or minor
  2. Inform students about technical communication classes


Personas

My personas are very narrow in scope. The technical communication major webpage is just a tiny part of the English Department website, which is just a tiny part of the entire Iowa State University website.

I got data for "The Declared Major" and "The Minor" persona from a survey I sent out to students on the technical communcation mailing list for a previous project where I created a data display about current technical communication students.

I got data for "The Undecided Major" persona when I was a TA for freshman and sophomore students in a technical communication learning community.





Stakeholder Interviews

View the Stakeholder Report

Process

To learn more about what the stakeholders wanted from the website redesign two stakeholders:

Samantha Adams - technnical communication advisor and website owner.

Dr. Maggie LaWare - program coordinator for the rhetoric graduate program and a decision maker in the English Department website

Findings

  • Both stakeholders wanted to use the website as a recruitment tool for new students as the main goal
  • Both stakeholders thought the visual design needs the most work
  • Samantha wanted users to learn more about the technical side of the major, while Dr. LaWare wanted to see more information about courses

Accessibility Testing

Major Problems

Remove or Replace the Image
The image on the page, see Image 1 in section 6.1, caused two failures. Removing the image and replacing it with a non-text based, purely decorative image. This will solve:

  • 1.1.1 Non-text Content (Level A)
  • 1.4.5 Images of Text (Level AA)

Add a focus element around hamburger menu buttonWhen the screen is narrower, the hamburger menu in the top navigation does not have a visible focus element to indicated that it can be opened or it is selected when a user is using a keyboard to navigate a website. Adding a visible focus element, like the blue box boarder, will solve this problem. This will solve:

  • 2.4.7 Focus Visible (Level AA)


Minor Problems

Make the page title shorter
Changing the title to “Technical Communication Requirements” will make the tile of the webpage shorter and cause less confusion about what the webpage content is explaining. It will also make it more consistent with the other English Department majors’ webpages. This will solve:

  • 2.4.2 Page Titled (Level AA)


Usability Tesing

View the Usability Testing Report

Three students (one freshman, one sophomore, one junior) were recruited for testing from the Writing Opportunities in the Work Place learning community. Two high school seniors were recruited from my hometown high school.

Quantitative Findings

Task 1:
Find the number of credits needed for the concentraded study area.

Success Rate Average Time
100% 20.2 seconds

Task 2:
Find if an internship is required to graduate.

Success Rate Average Time
60% 20.6 seconds


Task 3:
Find the semester it is recommended to take English 314.

Success Rate Average Time Average Clicks
70% 72 seconds 2.6 clicks


Task 4:
Find if English 350 is required for a Technical Communication Minor.

Success Rate Average Time Average Clicks
20% 82.8 seconds 5.6 clicks

Qualitative Findings

  • None of the testers understood what technical communication was after reading the webpate
  • None of the testers wanted to read any of the text
  • All testers wanted more images of students
  • 3 testers thought is would be helpful to see what users were doing for internships or after graduation


Prototype (version 1)

My First Prototype

Changes

  1. The first paragraph explaining what technical communication is was rewritten so it would make more sense to users
  2. The "Theory and History" and "Principles, Practices, and Technologies" were organized into two columns to make them easier to read and the page shorter
  3. The couse information was updated and organized into tabs with headeds explaining the types of classes
  4. An internship section was added, showing three students and information about their internships from the previous summer
